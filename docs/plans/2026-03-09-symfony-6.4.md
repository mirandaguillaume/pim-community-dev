# Phase 1: Symfony 5.4 → 6.4 LTS — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Upgrade all Symfony components from 5.4 to 6.4 LTS, fixing all breaking changes and deprecations.

**Architecture:** Pre-migration fixes first (while still on SF 5.4 to keep tests green), then composer dependency swap, then post-migration fixes. Each task is independently committable.

**Tech Stack:** PHP 8.2, Symfony 6.4, Doctrine ORM 2.13, Docker (`docker-compose run --rm php`)

**Previous analysis:** `docs/plans/2026-02-27-phase2-symfony64.md` — inventory of all blockers.

## Task 9: Fix Gedmo annotation reader configuration

Gedmo doctrine extensions use `@annotation_reader` service which is deprecated. In SF 6.4
with `doctrine/annotations` 2.x, this should be migrated to attribute mapping.

**Files:**
- Modify: `config/services/gedmo_doctrine_extensions.yml`
- Modify: `config/packages/doctrine.yml` (line 35)

**Step 1: Update Gedmo listener config**

In `config/services/gedmo_doctrine_extensions.yml`, replace the entire file:
```yaml
services:
    gedmo.listener.tree:
        class: Gedmo\Tree\TreeListener
        tags:
            - { name: doctrine.event_subscriber, connection: default }
        calls:
            - [ setAnnotationReader, [ '@Gedmo\Mapping\Driver\AttributeReader' ] ]

    gedmo.listener.timestampable:
        class: Gedmo\Timestampable\TimestampableListener
        tags:
            - { name: doctrine.event_subscriber, connection: default }
        calls:
            - [ setAnnotationReader, [ '@Gedmo\Mapping\Driver\AttributeReader' ] ]
```

Note: `Gedmo\Mapping\Driver\AttributeReader` is available since `gedmo/doctrine-extensions` v3.5.
If the installed version doesn't have it, an alternative is to keep `@annotation_reader` and
ensure `doctrine/annotations` is still installed. Check with:
```bash
docker-compose run --rm php composer show gedmo/doctrine-extensions | head -5
```

If version < 3.5, keep `@annotation_reader` and skip this step.

**Step 2: Update Gedmo mapping type in doctrine.yml**

In `config/packages/doctrine.yml` line 35, change:
```yaml
type: annotation
```
→
```yaml
type: attribute
```

Only do this if Gedmo >= 3.5 AND the Gedmo entities actually use PHP 8 attributes.
If they still use annotations, keep `type: annotation` and ensure `doctrine/annotations` is installed.

**Step 3: Verify cache clears**

```bash
docker-compose run --rm php php bin/console cache:clear --env=dev
```

**Step 4: Commit**

```bash
git add config/services/gedmo_doctrine_extensions.yml config/packages/doctrine.yml
git commit -m "refactor: migrate Gedmo extensions from annotations to attributes"
```
